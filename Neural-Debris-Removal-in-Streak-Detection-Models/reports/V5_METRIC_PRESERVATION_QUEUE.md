## Metric fact that changes the strategy

The public maCADD implementation filters only the hidden clean model's detections
at confidence `> 0.20`. It does **not** filter participant detections at `0.20`.
Therefore, a submitted box at confidence `0.03-0.15` can still match a hidden
clean box.

This makes deletion unusually risky:

- deleting a hidden-clean match pays its full hidden-clean confidence;
- retaining a low-confidence matching box avoids the unmatched false-negative
  term;
- a low-confidence box with no overlap costs only its submitted confidence.

The hidden clean predictions remain unavailable, so confidence floors must be
selected using the public-unlearn pseudo-clean metric and the frozen retention
gate, never from test predictions or leaderboard feedback.

## Frozen gates

A candidate is promotable locally only when all hold:

- poison fire rate at 0.20 `<= 0.35`;
- median poison score ratio `<= 0.25`;
- retained box match rate `>= 0.90`;
- median retained confidence ratio in `[0.80, 1.20]`.

Passing candidates are ranked by the exact public maCADD code against the
public-only pseudo-clean reference. That proxy is not presented as the hidden
leaderboard metric.

## Priority

1. E43-E46: inexpensive, metric-aligned output candidates.
2. E47-E48: low-dimensional model/structure candidates.
3. E49-E50: training candidates only if the cheap frontier does not pass.

## Research ideas deliberately not promoted

- Test-time pseudo-labeling or test-derived calibration: prohibited by Rule 7.A.
- Large poison-gate MLPs: only 20 poison source images, too much overfit risk.
- More aggressive generic pruning: the 60-channel causal audit already found
  no safe pruning candidate.
- Two-stage RPN/head inconsistency repair: RetinaNet is a one-stage detector, so
  the published two-stage module reset is not directly transferable.
- Diffusion trigger inversion: external generative models and a high-compute
  pixel search add rule and validation risk without addressing the observed
  semantic streak family.
